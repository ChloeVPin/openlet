# Security Policy

## Supported Versions

We actively maintain and provide security updates for the latest version of Openlet running on `master`.

| Version | Supported |
| ------- | --------- |
| Main (`master`) | :white_check_mark: |
| Older commits | :x: |

## Reporting a Vulnerability

We take the security of Openlet seriously. If you believe you have discovered a security vulnerability, please report it to us responsibly.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to Report

- **Email**: Send an email to [security@openlet.com](mailto:security@openlet.com).
- Include detailed steps to reproduce the issue, proof-of-concept (PoC), and potential impact.
- We will acknowledge receipt of your report within 48 hours and provide updates as we work to address the issue.

### Preferred Information
To help us triage and fix the issue quickly, please include:
- Type of vulnerability (e.g., XSS, Access Control / Authorization Bypass, Rate Limit Bypass, Resource Exhaustion).
- Affected location (e.g., specific API route, page, or server function).
- Step-by-step instructions or scripts to reproduce the issue.

### Security Expectations & Disclosure Policy
- Please allow us a reasonable timeframe to remediate the vulnerability before disclosing it publicly.
- Do not attempt to access, modify, or destroy user data during testing.
- Test only against local instances or designated non-production environments whenever possible.

Thank you for helping keep Openlet and its community safe!
